# Release notes

[!DNL Federated Audience Composition] continuously delivers new features, enhancements to existing features, and bug fixes. All changes are consolidated in these release notes. [!DNL Federated Audience Composition] is built natively on [!DNL Adobe Experience Platform] and inherits from its latest innovations and improvements. Learn more about these changes in [Adobe Experience Platform Release Notes](https://experienceleague.adobe.com/docs/experience-platform/release-notes/latest.html){target="_blank"}.

## January '26 release {#fac-26-01}

The January release for Federated Audience Composition supports the following new capabilities and improvements:

### New capabilities {#fac-26-01-feature}

| Azure Synapse Service Principal Authentication support |
| --- |
| You can now connect to Federated Audience Composition with Azure Synapse using a Service Principal. For more information about creating a connection, please read the [connections overview](../connections/home.md#create). |

| Availability for Adobe Experience Platform customers on Amazon Web Services (AWS) |
| --- |
| You can now use Federated Audience Composition if your Experience Platform instance is on AWS. For more information about Experience Platform on AWS, please read the [multi-cloud overview](https://experienceleague.adobe.com/en/docs/experience-platform/landing/multi-cloud). |

### Improvements {#fac-26-01-improvements}

This release comes with the following improvement.

- **Data expiration configuration for audiences**

    You can now set a data expiration when using the **Save audience** activity in a composition. 

    To learn more about data expirations in Federated Audience Composition, please read the [activities guide](../compositions/activities.md#save-audience).
